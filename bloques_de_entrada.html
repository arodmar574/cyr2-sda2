Bloques de entrada

BLOQUES DE ENTRADA

Los botones A y B son entradas que podemos usar para activar partes de nuestro programa. Estas partes se llaman manejadores de eventos y se ejecutan cuando se cumple una condición, como por ejemplo, pulsar un botón. Podemos programar el manejador para que responda al botón A, al B o a ambos a la vez. Si usamos un navegador, simularemos los botones en la pantalla en lugar de los del micro:bit.

  • Para el botón A o B: El manejador se activa cuando el botón se presiona y se libera rápidamente, en menos de un segundo.
  • Para A y B juntos: El manejador se activa cuando los dos botones se presionan al mismo tiempo. Tenemos un margen de un segundo y medio desde que presionamos el primer botón hasta que presionamos el segundo para que se considere que los pulsamos simultáneamente.

El ejemplo muestra cómo llevar un registro de las veces que se presiona el botón A. Cada vez que se hace, la variable de recuento aumenta y se muestra en la pantalla LED.

  • Para botón A o B presionado

El bloque verifica si hay algún botón que se está presionando en ese instante.

Puedes comprobar el funcionamiento de este bloque en este ejemplo:

  • Al pulsar el logotipo

El logo de micro:bit V2 funciona como un botón sensible al tacto, por lo que es un dispositivo de entrada que activa su código cuando lo tocamos y lo desactiva cuando lo dejamos de tocar.

  • El logotipo está pulsado

El logo de micro:bit es un botón que devuelve un valor booleano según si se está tocando o no. Este valor se puede usar para controlar el flujo del programa.

El ejemplo cambia la imagen que se muestra según el estado del logo. Si se toca el logo, se muestra una cara feliz. Si no se toca, se muestra una cara triste.

Reto 6: Enciende o apaga corazón con dos botones

Consiste en que cada vez que se pulsa el botón “A” el estado del corazón en la pantalla debe pasar de encendido a apagado, o de apagado a encendido. Al principio del programa el corazón debe estar apagado.

Contenidos trabajados:

  • Entorno de programación Makecode
  • Envío de contenidos a microbit
  • Trabajo inicial con eventos
  • Entradas con pulsadores

¿Necesitas el pseudocódigo?

  • Haz que al presionar A se muestra un corazón.
  • Haz que al presionar B desaparezca.

Reto 7: Piedra, papel, tijera

Esta actividad se trata de agitar microbit para que muestre el símbolo de piedra, papel o tijera al azar para jugar contra un humano.

Contenidos trabajados:

  • Entorno de programación Makecode
  • Envío de contenidos a microbit
  • Condicionales
  • Creación y uso de variable
  • Trabajo inicial con eventos
  • Entradas con acelerómetro (eventos)
  • Condicionales (si)
  • Usa el bloque de agitar.
  • Crea la variable "número"
  • Utiliza los condicionales si, sino si y si no.
  • Asigna la piedra, papel o tijera a números.

Reto 8: Dado electrónico

Se trata de un dado electrónico, cada vez que agitamos microbit nos muestra los números al azar como en un dado.

Contenidos trabajados:

  • Entorno de programación Makecode
  • Envío de contenidos a microbit
  • Condicionales
  • Creación y uso de variable
  • Trabajo inicial con eventos
  • Entradas con acelerómetro (eventos)

Condiciones del ejercicio:

  • Usa el bloque de agitar para agitar el dado.
  • Crea la variable "número".
  • Recuerda que un dado tiene 6 caras.
  • Asigna un número para cada cara del dado mediante condicionales si, sino, si y si no.
